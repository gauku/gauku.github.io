    • Types of questions:
    1. Anomaly detection - is the machine behaving weirdly
    2. Fault classification / root cause isolation- is there a fault. if yes, which one?
    3. Remaining useful life estimation
    Types of data involved - Images, sensors. which to use depends on device + type of question that you wanna answer.

    Challenges: (Major takeaway - need to work closely with engineers (domain/machine experts))
    1. Data acquisition - how do you get failure data when every company is trying to avoid exactly that.
      • Need domain expert engineers to check the right kind of the failure data. Their knowledge of what failure data might look like needs to be incorporated in the predictive model
      • Might need signal processing techniques, model-based techniques(?), modeling the system dynamics etc.
      • Companies often build models/simulations of their systems during the design stages which can be used by injecting failure modes and faults into them and generate synthetic (but cheap and safe, but also not really real o.O ) failure data.
    2. Feature extraction and reduction - any data you might get your hands onto, you'll have lots of it.. think GBs and TBs each day. Cost and time efficient methods of doing this. Reduction still must retain all the useful information
    3. Data science - method, training, and your usual ML questions belong here
    4. and lastly, integration into production. Do you go cloud, on premise, hybrid, IoT/edge devices
